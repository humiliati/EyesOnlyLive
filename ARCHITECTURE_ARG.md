## Key Architectural Decisions

### 1. Two-Tier Persistence
- **Registry**: In-memory with lazy loading from multiple sources
- **KV Storage**: Persistent data for events, drops, inventories

### 2. Event-Driven Updates
- Components subscribe to sync manager
- Real-time updates without polling UI
- Clean separation of concerns

### 3. Modular Item Sources
- Canonical base (static JSON)
- Live ARG events (KV)
- Optional external API
- All merged into single registry

### 4. Grid + GPS Dual System
- Grid (A-H, 1-8) for tactical display
- GPS (lat/lng) for precise positioning
- Both stored, either used

### 5. Separation from Equipment
- ARG items = Collectibles (simple)
- Equipment = Full assets (complex)
- Different UI, same map

### 6. M Console Authority
- All creation flows in M Console
- Agents can discover/transfer (future)
- Clear operator control model
